# Πολυδιάστατες Δομές Δεδομένων - TMDB Movies Analysis

## Μέρος 1: Οδηγίες Εκτέλεσης και Demo

Αυτό το project υλοποιεί και συγκρίνει διάφορες πολυδιάστατες δομές δεδομένων (KD-Tree, QuadTree, Range Tree, R-Tree) σε συνδυασμό με Locality Sensitive Hashing (LSH) για την αναζήτηση ταινιών από το dataset της TMDB.

### Προαπαιτούμενα
Όλες οι απαραίτητες βιβλιοθήκες βρίσκονται στο αρχείο `requirements.txt`. Μπορείτε να τις εγκαταστήσετε αυτόματα με την εντολή:
```bash
pip install -r requirements.txt
```

### Εκτέλεση Demo (Κεντρική Αξιολόγηση)
Το κύριο αρχείο εκτέλεσης που πραγματοποιεί όλα τα πειράματα και τις συγκρίσεις είναι το:
**`comprehensive_evaluation.py`**

Για να το εκτελέσετε, τρέξτε στο τερματικό:
```bash
python comprehensive_evaluation.py
```
**Τι κάνει αυτό το script:**
1. Φορτώνει και φιλτράρει τα δεδομένα από το `data_movies_clean.csv`.
2. Κατασκευάζει όλους τους πολυδιάστατους δείκτες (KD-Tree, κλπ).
3. Εκτελεί σύνθετα ερωτήματα "Range Search + LSH" (5-διάστατη αναζήτηση συν ομοιότητα εταιρειών παραγωγής).
4. Αποθηκεύει τα αποτελέσματα στον φάκελο `evaluation_results/` σε μορφή JSON.
5. Εμφανίζει στην οθόνη τους μέσους χρόνους κατασκευής και αναζήτησης.

### Οπτικοποίηση και EDA
Για να δείτε τα αναλυτικά γραφήματα και την Εξερευνητική Ανάλυση Δεδομένων (EDA), μπορείτε να ανοίξετε το Jupyter Notebook:
**`evaluation_with_eda.ipynb`**

### Δομή Αρχείων
- `kd_tree.py`, `r_tree.py`, `range_tree.py`, `quadtree.py`: Υλοποιήσεις των δομών.
- `lsh.py`: Υλοποίηση του MinHash LSH για κείμενο.
- `project1_loader.py`: Φόρτωση και αρχική επεξεργασία του CSV.
- `preprocess_text.py`: Tokenization εταιρειών παραγωγής.
- `extract_5d_vectors.py`: Εξαγωγή των 5 αριθμητικών διαστάσεων.

## Μέρος 2: Τεχνική Περιγραφή και Αποτελέσματα

Πλήρης Περιγραφή Δεδομένων
Το σύνολο δεδομένων προέρχεται από το αρχείο data_movies_clean.csv και περιέχει 946.460 εγγραφές ταινιών από το 1900 έως σήμερα. Περιλαμβάνει πληροφορίες για την παραγωγή τη γλώσσα τη χώρα προέλευσης καθώς και μετρικές δημοτικότητας και βαθμολογίας.

Ορισμοί Όλων των Στηλών
- id: μοναδικός αναγνωριστικός αριθμός ταινίας
- title: ο επίσημος τίτλος της ταινίας
- adult: ένδειξη αν η ταινία είναι για ενήλικες
- original_language: η αρχική γλώσσα παραγωγής
- origin_country: η χώρα ή οι χώρες προέλευσης
- release_date: ημερομηνία πρώτης κυκλοφορίας
- genre_names: λίστα με τα είδη της ταινίας
- production_company_names: εταιρείες παραγωγής
- budget: προϋπολογισμός σε δολάρια
- revenue: συνολικά έσοδα σε δολάρια
- runtime: διάρκεια της ταινίας σε λεπτά
- popularity: δείκτης δημοτικότητας στην πλατφόρμα
- vote_average: μέση βαθμολογία από τους χρήστες
- vote_count: πλήθος ψήφων που καταγράφηκαν

Αποτελέσματα Χρόνου Εκτέλεσης
- kdtree: Χρόνος κατασκευής 1.97s και Χρόνος αναζήτησης εύρους 0.04s
- quadtree: Χρόνος κατασκευής 7.37s και Χρόνος αναζήτησης εύρους 0.42s
- rangetree: Χρόνος κατασκευής 210.27s και Χρόνος αναζήτησης εύρους 4.43s
- rtree: Χρόνος κατασκευής 73.20s και Χρόνος αναζήτησης εύρους 36.07s

Σχολιασμός Χρόνων
Το KD-Tree είναι η πιο γρήγορη δομή τόσο στην κατασκευή όσο και στην αναζήτηση εύρους για αυτό το σύνολο δεδομένων. Το Range Tree παρουσιάζει τη μεγαλύτερη καθυστέρηση στην κατασκευή λόγω της πολυπλοκότητας των εσωτερικών του δομών. Το R-Tree είναι αρκετά αργό στις αναζητήσεις εύρους σε σύγκριση με τις άλλες διαθέσιμες επιλογές.

Αποτελέσματα Κλιμάκωσης για 100.000 Στοιχεία
- KD-Tree: Χρόνος κατασκευής 0.99s και Χρόνος αναζήτησης KNN 0.12s
- QuadTree: Χρόνος κατασκευής 3.96s και Χρόνος αναζήτησης KNN 108.67s
- R-Tree: Χρόνος κατασκευής 39.48s και Χρόνος αναζήτησης KNN 160.75s

Σχολιασμός Κλιμάκωσης
Η απόδοση του KD-Tree παραμένει εξαιρετική ακόμα και με 100.000 στοιχεία δείχνοντας πολύ καλή κλιμάκωση για αναζητήσεις γειτόνων. Το QuadTree και το R-Tree εμφανίζουν σημαντική επιβράδυνση στις λειτουργίες KNN όταν ο όγκος των δεδομένων αυξάνεται δραματικά. Η διαφορά στην ταχύτητα κατασκευής παραμένει σταθερή με το KD-Tree να υπερέχει σημαντικά των υπολοίπων δομών.

Αναλυτική Αξιολόγηση QuadTree
- Μέγεθος 50.000: Ύψος δέντρου 4 και Χρόνος αναζήτησης 1.54ms
- Μέγεθος 200.000: Ύψος δέντρου 5 και Χρόνος αναζήτησης 4.36ms
- Μέγεθος 500.000: Ύψος δέντρου 5 και Χρόνος αναζήτησης 9.05ms

Σχολιασμός QuadTree
Το ύψος του QuadTree παραμένει πολύ χαμηλό ακόμα και για μισό εκατομμύριο στοιχεία γεγονός που ευνοεί την ταχεία πλοήγηση στις περιοχές του χώρου. Ο χρόνος αναζήτησης αυξάνεται γραμμικά με το μέγεθος του συνόλου δεδομένων χωρίς να προκαλεί ακραίες καθυστερήσεις στις βασικές λειτουργίες. Η δομή αποδεικνύεται αξιόπιστη για την οργάνωση μεγάλου πλήθους σημείων σε πολυδιάστατο περιβάλλον.

Αποτελέσματα Μνήμης για 500 Στοιχεία
- KD-Tree: Χρήση μνήμης 0.19 MB
- QuadTree: Χρήση μνήμης 0.48 MB
- Range Tree: Χρήση μνήμης 294.02 MB
- R-Tree: Χρήση μνήμης 0.30 MB

Σχολιασμός Μνήμης
Το KD-Tree είναι η πιο αποδοτική δομή όσον αφορά τη χρήση μνήμης απαιτώντας τον λιγότερο χώρο για την αποθήκευση των δεδομένων. Το Range Tree έχει εξαιρετικά υψηλές απαιτήσεις μνήμης καθιστώντας το ακατάλληλο για πολύ μεγάλα σύνολα δεδομένων στην τρέχουσα υλοποίηση. Οι δομές QuadTree και R-Tree έχουν παρόμοια και χαμηλή κατανάλωση μνήμης για το μέγεθος των 500 στοιχείων.
